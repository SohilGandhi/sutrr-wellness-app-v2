# UI/UX Glossary & Terminology

> **Purpose**: A living dictionary of design principles, frameworks, and jargon used in this project.
> **Status**: INTERNAL ONLY (Ignored by Git)

| Term / Concept | Category | Definition & Context | Example in Our App |
| :--- | :--- | :--- | :--- |
| **Glassmorphism** | Visual Style | A design trend emphasizing translucency (frosted glass effect), vivid background colors, and light borders. | Bottom Navigation Bar, Dashboard Cards (Premium feel). |
| **Haptic Feedback** | Interaction | Tactile response (vibration) when a user interacts with UI elements. | (Planned) When completing a wellness check-in. |
| **Progressive Disclosure** | UX Principle | Show only necessary information first, and reveal more as the user interacts. Prevents cognitive overload. | "Read More" in Articles, Expandable Product Details. |
| **F-Pattern** | Layout | Users scan screens in an 'F' shape (Top left to right, then down). | Dashboard layout: Header (Top) -> Wellness Score (Left/Center) -> Quick Actions. |
| **Thumb Zone** | Mobile UX | The most comfortable area for one-handed thumb interaction (bottom half of screen). | Bottom Navigation, Floating Action Buttons, Primary CTAs. |
| **Skeleton Loading** | Performance | A placeholder layout (gray boxes) shown while data is loading. Reduces perceived wait time. | (Planned) Shop page product grid loading state. |
| **Micro-interaction** | Interaction | Subtle animations that provide feedback (e.g., button press, toggle switch). | Heart icon animation, Toggle switches in Profile. |
| **DPDP Act** | Compliance | *Digital Personal Data Protection Act (India)*. Requires explicit consent and data minimization. | Age Gate, Consent Toggles, Data Delete option. |
| **ABHA** | Compliance | *Ayushman Bharat Health Account*. India's digital health ID system. | "Link ABHA ID" banner in Experts tab. |
| **Dark Patterns** | Anti-Pattern | Deceptive design tricks to make users do things they didn't intend (avoid these!). | We strictly avoid pre-ticked consent boxes or hidden unsubscribe buttons. |
| **Negative Space** | Visual Design | Empty space around elements (Whitespace). Improves readability and premium feel. | "Spacious V2" redesign of Product Cards. |
| **Call to Action (CTA)** | UI Element | The primary button causing a user action. | "Add to Cart", "Book Session", "Start Quiz". |
| **Destructive Action Modal** | UX Pattern | A confirmation popup shown before irreversible actions (delete, erase). Uses red buttons + clear consequence language. | Delete Entry confirmation in Journal. |
| **Feature Discoverability** | UX Principle | Making features easy to find through text labels, prominent placement, and visual cues (not just icons). | "Ask AI" labeled button in Dashboard header, persistent nav text labels. |
| **Consent Artifact** | Compliance | A UI element that captures explicit, informed consent per DPDP 2023 before processing personal data. | Share-to-AI consent modal in Journal with privacy notice. |
| **Purpose Limitation** | Compliance | DPDP principle: data can only be used for the specific purpose for which consent was obtained. | AI consent modal states: "Data will only be used for this conversation." |
| **Touch Target** | Mobile UX | Minimum interactive element size for comfortable tapping. WCAG 2.1 AA requires 44px; we use 48px. | All buttons, nav items, input fields. |
| **Bottom-Sheet Drawer** | UI Element | A panel that slides up from the bottom of the screen. Ideal for mobile filters and settings. | Shop Filters, Expert Profiles. |
| **Telemedicine Guidelines (2020)** | Compliance | Indian Govt rules requiring doctors to show credentials, RegNo, and emergency disclaimers for teleconsults. | Expert profile drawer disclaimer. |
| **Quick Exit / Panic Button** | UX Pattern | A discreet button allowing users to instantly leave sensitive pages without leaving browser history traces. | "X" icon in Experts and Dashboard headers. |
| **Social Proof Ticker** | Conversion | An auto-updating banner showing peer activity to build trust. | "3 doctors online" on Dashboard. |
*This list is dynamic. Add new terms as they are encountered.*
